Contributing

Background

The master branch stays like the upstream Bootstrap master and acts as a vendor branch. The bizo branch contains our changes (bizstrap).

E.g. at any time we can git diff origin/master origin/bizo to see what we've changed vs. Bootstrap proper.

Versioning

Bizstrap version numbers are bootstrap version.our version.

E.g. 2.0.3.1 is Bootstrap 2.0.3, with our 1st release.

Making Changes

  1. Make your changes to the bizo branch, which has our changes
  2. When changing existing Bootstrap files, make changes inline, and leave a comment with the reason for your change. This should facilitate merging in changes in the future, as we'll get explicit conflicts (vs. changes in separate files) and also know why we had changed what we did.
  3. Sync your changes git push origin bizo
  4. Update code.bizo.com/bizstrap (aka this site) by running sync-docs-from-bizo-branch.sh in the gh-pages branch, then git push origin gh-pages.

How to build Bizstrap

Dependencies

  • You need to install lessc, the less cli compiler, see instructions: here. You will probably need to install node.js
  • You need to have Ruby 1.8.7 or higher installed ruby -v
  • To compile the docs you need to have the jekyll gem installed gem install jekyll

Compiling Less

Simply run ./bin/build.rb docs/assets/css/bootstrap.css

Compiling Docs

From the bizo branch cd jekyll_docs && jekyll. If you want to see updates as you change files use jekyll --server --auto
