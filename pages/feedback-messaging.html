Design Patterns for Feedback Messaging

Defined here are a few of the design patterns for providing feedback messaging in BMP, as well as proposed content.

Sticky Alert

Example Sticky Alert

When to use

  • To confirm an action a user has just taken
  • To display a limitation that would normally be shown in-line but can't be due to page layout (in a report table, for example)

Specifications

Use the alert-sticky-backdrop, alert-sticky, and alert-inverse classes as shown below to create an appropriate-looking alert message that's fixed to the top of the viewport. The alert-inverse class was created just for this purpose, so try not to use the other alert styles with this pattern.

Use CSS transitions to fade the message in, display it for 2 seconds, and then fade it out.

<div class="alert-sticky-backdrop">
  <div class="alert alert-inverse alert-sticky">
    <p><i class="icon-ok icon-white"></i> Share added to Brand Bar display rotation</p>
  </div>
</div>

Inline Messaging

BMP supports inline field validation and should validate fields upon completion. See the Forms doc for more info.

Inline messaging may also use the Contextual Help patterns and the standard Bootstrap alert classes depending on the case.
