Bizstrap Quickstart

For any Bizo app just copy and paste the following code below

            
<!DOCTYPE HTML>
<html>
  <head>
    <link href="http://media.bizo.com/bizstrap/css/bizstrap-v2.0.3.css" rel="stylesheet">
  </head>

  <body class="bizo-app">
    <div class="container-fluid">
      <div class="row-fluid">

        <!-- Sidebar  -->
        <div class="span2 content"></div>

        <!-- Main Content  -->
        <div class="span10 content"></div>
      </div>
    </div>

    <!-- JavaScript for widgets & charts  -->
    <script src="http://ajax.googleapis.com/ajax/libs/jquery/1.7.2/jquery.min.js"></script>
    <script src="https://raw.github.com/DmitryBaranovskiy/raphael/master/raphael-min.js"></script>
    <script src="https://raw.github.com/jcarver989/raphy-charts/master/compiled/charts.min.js"></script>
  </body>
</html>

Bizstrap Source Conventions

  • The master branch stays like the upstream Bootstrap master and acts as a vendor branch.

    E.g. at any time we can git diff origin/master origin/bizo to see what we've changed vs. Bootstrap proper.

  • The bizo branch has our changes

  • Bizstrap version numbers are bootstrap version.our version.

    E.g. 2.0.3.1 is Bootstrap 2.0.3, with our 1st release.

  • When changing existing Bootstrap files, make changes inline, and leave a comment with the reason for your change.

    This should facilitate merging in changes in the future, as we'll get explicit conflicts (vs. changes in separate files) and also know why we had changed what we did.

  • To update code.bizo.com/bizstrap, make changes in the bizo branch, then git push origin bizo:gh-pages.

    This means we don't have to maintain both bizo and gh-pages branches, but satisifies GitHub, which only looks for the gh-pages branch.
